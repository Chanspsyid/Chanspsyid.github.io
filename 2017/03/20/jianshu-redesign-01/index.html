简书 - 关注模块重设计：设计目标

发表于 2017-03-20   |     |   阅读次数

问题：如何确定关注模块的设计目标？

回答角度：从功能定位、用户需求这两个角度来探索设计目标。

简书的产品定位

  • 聚集创作者并鼓励创作和分享，构建最大的原创内容社区

简书的产品特点

  • 创作目标明确，工具性比较强
  • 独立的全类别内容产品，质量层次不齐，尾部内容丰富，缺少头部内容
  • 基于兴趣的开放性关系链，有分享和社交的性质
  • 内容的传播和持续曝光能力较差

简书的产品价值

  • 创作者群体、流量和数据
  • 读者群体、流量和数据
  • 内容

产品策略（长期／短期）：为利益（当下收益）还是为未来（吸引投资）

创作者策略

  • 持续吸引更多的创作者进入简书，扩大创作群体：讲好创作故事。
  • 吸引并留住最重要的创作者。
  • 通过产品设计、内容运营、社交互动等提高内容的曝光能力，用来激励创作。
  • 通过创作支持、变现支持、运营和社交激励等方式来鼓励创作和分享，帮助创作者沉淀内容、自我成长。
  • 保证创作体验，提高产品的易用性。

读者策略

  • 持续吸引更多的读者进入简书：高质量创作者、感兴趣的内容、创作和成长故事。
  • 帮助读者发现更多感兴趣的创作者和内容
  • 设计体验良好的关系链内容和过滤器，提升读者的内容体验
  • 鼓励阅读和分享
  • 鼓励加入创作
  • 鼓励形成读者社群，通过社交、共同兴趣来增强产品黏性

内容策略

  • 以兴趣为导向
  • 以标签和专题为内容沉淀的载体
  • 鼓励头部内容的产出和沉淀
  • 曝光、传播和分享，增强内容的流动效率和体验
  • 加入社交因素，活跃气氛、增加创作灵感和素材

关系链策略

  • 增强创作者与读者之间的联结和黏性
  • 鼓励付费和社交，增强创作者和读者之间的双向激励

变现策略

  • 创作者和读者的关系链服务
  • 品牌内容合作
  • 以用户研究和数据、内容数据提供相关决策咨询
  • 创作者成长服务
  • 创作者变现服务
  • 自建 IP 和 IP 开发
  • 周边电商
  • 付费阅读
  • 垂直读者成长服务

关注模块的功能定位和目标是什么？

关注模块：关系链内容 + 关系链

推送关系链内容：区别于长尾随机内容，这里向读者推送已经被其筛选出来的内容源，帮助读者获取感兴趣的内容。

强化关系链联结：文章不错，我关注了对的人；没错，就这样给我更多的反馈吧，我会更加努力创作的。

关注模块的用户需求是什么？

目标用户

  • 对简书的写作方式和氛围有好感，希望获取创作反馈、偶尔浏览一下相关领域内容的普通创作者
  • 有明确的喜好、关注了一部分优质写手、不定期打开简书看一下已关注内容的读者

使用场景

  • 地铁／车里
  • 吃饭时
  • 工作空闲时
  • 睡前／早起

用户目标

  • 浏览最近大家都在看什么内容，看看趋势，找找谈资和灵感
  • 关注的作者和专题里有什么值得看的内容
  • 找找跟自己相关的东西，有没有什么可以启发自己的内容和作者

关注模块的设计目标是什么？

内容需求：圈子趋势、有看点的推送、跟自己有关的作者和内容

设计目标：快速、直接、积极、有意义
