Update

发表于 2016-06-01   |     |   阅读次数

博客很久没有使用，今天集中维护下，添加了一些小的优化。同时，本文将作为维护日志保持更新。

2016.06.01：还有两个坑没填

博客架构更新：

更新 hexo：

npm update -g hexo

可能会要求 root 权限，之后输入密码即可：

sudo -s

更新主题：cd 到主题文件夹，执行命令

git pull

更新插件：

npm update

修复标签页面

由于标签页面中 Markdown 语法错误，导致无法显示，解决了两个问题：

type: "tags" #类型名的冒号后空一格

--- #页面定义结束后的分割线格式

移动端的响应问题

昨天发布完文章，在手机上打开后，发现页面主题适应了屏幕，但页面里的内容没有：

截图示例

猜想可能是架构版本 bug，因为刚建完博客并没有这个问题。但今天更新完博客架构和主题后，问题依然存在。

检查 config 文件，没有发现特别的地方。

修改主题 Mist 为 Pisces后，问题解决。

再次换回 Mist，多次清理本地静态文件，还是不行，难道是上传的时候丢失？

已提交 issue。

首页只显示一篇文章的问题

这个问题很让人疑惑，因为有几次什么都没有做，重新部署后就恢复了正常。

尝试重新更新主题文件后，发现提示本地的 config 文件为了防止丢失设置，并没有一起更新，备份并删除后，再次更新。

结果不仅解决了这个问题，还解决了移动端的响应。。。

而且主题样式还有了更新。。。

真是明珠暗投啊，这么好的主题被我瞎折腾出这么多“问题”。。。

要认真读代码和文档啊！

百度搜索的提交略麻烦，需要验证，我使用的是 CNAME 验证。

但疑惑的是，百度搜索提交后也搜不到网站。

如果有自己的 Google Analysis 账号，这一步就简单多了。

配置后在 public 文件夹中可以分别将 sitemap.xml 和 baidusitemap.xml 提交给谷歌和百度。

提交到谷歌后，发现读取错误：URL not allowed，同时右侧的网址示例都是这样的：

http://yoursite.com/tags/index.html

这个问题出在站点配置文件，进入 hexo 的 config.yml，将 url 后的 http://yoursite.com 改为自己的站点地址，就可以搞定了。

但百度这边还会出现这样的情况：

FATAL duplicated mapping key at line 88, column 23
path: baidusitemap.xml

我还找不到这句语法错在哪里，只能删掉。不过，就算删掉这句，baidusitemap.xml 该出来还是会出来，可能是之前装的插件弄的吧。。。

百度站点地图提交的位置隐藏较深，在［网页抓取］下的［链接提交］页面的下半部分的［sitemap］标签里，提交结束后，还可以检测并更新［robots］。

不过，百度统计在提交成功后，显示的是抓取失败，使用百度也搜不到博客的地址。等一段时间再看看吧。

增加阅读量统计

根据 Next 文档的介绍，选择 Leancloud 方式记录阅读量，建立了账户后，直接新建任意命名的应用，并点击［存储］页面的数据右上角的设置，新建 Class，用于专门保存博客阅读量的数据，Class 命名为 Counter。

点击顶部导航的设置，进入［应用 key］，就可以看到所需要的 App ID 和 App Key，复制并在NexT主题的 config.yml 文件中相应的位置填入即可：

重新部署博客，需要特别说明的是，记录文章访问量的唯一标识符是文章的［发布日期］以及［文章的标题］，因此请确保这两个数值组合的唯一性，如果你更改了这两个数值，会造成文章阅读数值的清零重计。

令附上需要注意的后台管理和隐私安全问题，在原文中都有提到。

个人由于姿势水平不足，无法独立设置 web 安全域名，先挖个坑。虽然没有完美攻略，不过还是成功完成这个任务。

社会化功能

修改主题配置文件中的 social 和 social_icons 字段，部署时 layout/.DS_Store 出错，本地预览发现右侧边栏的社会化部分溢出，应该是设计的自动换行，或者是 CSS 出了问题。

经过测试，社交链接是默认居中的，这样一看，没有 icon 还挺简洁的，先这样吧。

不过再挖个坑，编译的时候出这么多 error 太丑了。

可添加音乐播放器（网易云）

将单曲或歌单的外链扔进 markdown 里，如果不想自动播放，把代码中的 aotu 值从 1 改到 0 即可。太简单了。
